面试的时候有时候会听到这样的问题：

很多同学(包括我)往往因为不使用中间变量而卡壳了。网上流传了不使用中间变量实现swap的算法版本，大意如下, 注意， ^ 在C语言中是xor运算符：

分析

为什么swap2可行呢？我们来做如下的分析：

假设*a的最初值为$\alpha$ , *b的最初值为$\beta$ , 之所以用希腊字母，主要是因为小写字母已经用作表示变量了，只好用希腊字母来表示变量中存储的值，大家不必太在意。swap2函数第二行*b = *a ^ *b; 计算结果：

*b $\leftarrow \alpha$ xor $\beta$

第三行：*a = *a ^ *b; ,请注意到*b的值此时$\alpha$ xor $\beta$ ， 所以有

*a $\leftarrow \alpha$ xor ($ \alpha$ xor $\beta$)

$\alpha$ xor $\alpha$ 为0， 0 xor $\beta$ 为 $\beta$, 再赋值到*a， 这时候*a 就变成$\beta$了。

第四行*b = *b ^ *a; 此时*a已经是$\beta$ 了， *b 还是$\alpha$ xor $\beta$ ，所以第四行的代码实际赋值操作是：

*b $\leftarrow$ ($ \alpha$ xor $\beta$) xor $\beta$

这样*b值就赋值为$\alpha$ ,于是完成了交换操作。

依据上面的分析，我们可以立刻想到如下的算法：

这里实际运行过程如下：

void swap3(int *a, int *b) {

​ *b $\leftarrow \alpha$ - $\beta$
​ *a $\leftarrow \alpha - (\alpha$ - $\beta)$
*b $\leftarrow (\alpha - \beta) + \beta$

}

这里不考虑加法的溢出问题。

如果不用加减法，不用xor，还有没有类似的运算符可以实现不使用中间变量的swap算法呢？

答案是有的。只要这个运算是一个群 就可以了。

  • 群的定义如下：

​ 设S是一个非空集合， 而 $\cdot$ 是一个二元运算，

​ (1) * 运算如果满足结合律：

$a\cdot b\cdot c = a \cdot (b\cdot c) $ 则代数系统 是一个半群。

(2) * 运算有单位元e

$a \cdot e = e \cdot a = a$

(3) 每个元素有逆元

$\forall a \exists a^{-1}$ 使得 $ a \cdot a^{-1} = e $

所以这些算法实质如下：
void swap(int *a, int *b) {
*b = *a $\cdot (*b) ^{-1}$;

​ *a = *a $\cdot (*b) ^{-1}$;

​ *b = *b $\cdot (*a)$;
}

分析方法与加法和xor相同，可以看到，+运算与xor运算都是群。 在加法运算中，$a$ 的逆元为 $-a$, 单位元e为0。

同理，xor运算中，$a$ xor $a = 0$ ，0位单位元，$a$的逆元为$a$ .

所以，这里不光是xor和+ 运算可以使用，只要是群运算都是可以。

评估

我们来看看汇编之后的结果.

可以看到，使用xor 方式实现的swap与使用中间变量的swap相比，指令偏多，效率低。

结论

虽然不使用中间变量实现swap听起来十分高大上，但是实际收获不高（仅仅省一个字长的临时变量），而且性能较低（可以从反汇编中看到），使用场景也局限（加法的不能处理较大的整数，会有溢出问题，xor不能处理浮点数），纯粹为了考人而考人，社会主义建设中有许多问题等待解决，奉劝大家不要把精力放在这些无聊的题目上。
